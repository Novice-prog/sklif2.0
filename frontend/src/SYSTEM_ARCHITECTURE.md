# Архитектура системы учета клинических данных

## Концептуальная модель

### 5 ключевых принципов

1. **Пациент — только паспортные данные**
   - ФИО, дата рождения, пол, номер медицинской карты, ID ЕМИАС
   - Никаких изменяемых во времени данных (диагнозы, анализы и т.д.)

2. **Участие — эпизод в исследовании**
   - Связь пациента с конкретным экспериментом и контрольной группой
   - Один пациент может иметь несколько участий (разные исследования/группы)
   - Пример: Иванов участвует в Исследовании-А (группа 1) и через месяц — в Исследовании-B (группа 2)

3. **КТ_участия — контрольные точки внутри эпизода**
   - Конкретные визиты/наблюдения с фактическими датами
   - У одного участия может быть сколько угодно КТ
   - Основаны на шаблонах (КТ1, КТ2, выписка и т.д.)

4. **Факты привязаны к КТ**
   - **Диагноз_событие**: диагноз с ролью (Основной/Осложнение/Конкурирующее/Сопутствующее/Фоновое)
   - **Лаб_результат**: числовые/качественные значения анализов
   - **Инстр_результат**: данные инструментальных исследований
   - **Клин_состояние**: клинические показатели (АД, ЧСС, шкалы)
   - **Лечение_событие**: назначения (метод, доза, даты)

5. **Справочники отдельно**
   - Диагнозы, Биоматериалы, Показатели, Исследования, Методы лечения и т.д.
   - Не зависят от времени, переиспользуются

## Пример использования

### История пациента Иванова

**Пациент Иванов** → **Участие #1** в «Исследовании-А», Контр. группа 1

#### Три визита:
- **КТ#1** (При поступлении)
- **КТ#2** (До лечения)
- **КТ#3** (После лечения)

#### На КТ#1:
- **Диагнозы:**
  - «Отравление веществом Х» (тип: Основной)
  - «Пневмония» (тип: Сопутствующее)
- **Лабораторные:**
  - AST: 78 Ед/л (материал: Кровь)
- **Инструментальные:**
  - УЗИ брюшной полости: «без особенностей»
- **Клиника:**
  - АД: 150/95 мм рт.ст.

#### На КТ#2:
- **Диагнозы обновились:**
  - «Отравление веществом Х» (Основной)
  - «Острая печёночная недостаточность» (Осложнение)
- **Лечение:**
  - Инфузионная терапия: NaCl 0.9% 1000 мл

#### На КТ#3:
- Следующий срез данных...

Если Иванов включается в **Исследование-B** — это **Участие #2** со своей линейкой КТ.

## Структура данных

### Основные сущности

```
Пациент (Patient)
├── Участие (Enrollment) - связь с экспериментом и группой
    ├── КТ_участия (ControlPointInstance)
        ├── Диагноз_событие (DiagnosisEvent)
        ├── Лаб_результат (LabResult)
        ├── Инстр_результат (InstrumentalResult)
        ├── Клин_состояние (ClinicalState)
        └── Лечение_событие (TreatmentEvent)
```

### Справочники (References)

- **Эксперименты** и **Контрольные группы**
- **Шаблоны контрольных точек** (КТ1, КТ2, выписка...)
- **Диагнозы** (МКБ-10)
- **Биоматериалы** (кровь, моча и т.д.)
- **Исследуемые показатели** (AST, ALT, гемоглобин...)
- **Группы показателей** (биохимия, ОАК...)
- **Инструментальные исследования** (ЭКГ, УЗИ, рентген...)
- **Клинические признаки** (АД, ЧСС, температура...)
- **Биологические системы** (сердечно-сосудистая, дыхательная...)
- **Локусы** (печень, сердце, легкие...)
- **Методы лечения** (инфузионная терапия, гемодиализ...)

## Преимущества архитектуры

### 1. Временная динамика
Всё, что меняется, привязано к КТ_участия. График показателей по визитам — простая выборка.

### 2. Множественные исследования
Несколько «Участий» у одного пациента — данные не смешиваются.

### 3. Гибкие диагнозы
Вместо колонок «основной/сопутствующий» — таблица событий с ролью. Диагнозы меняются от КТ к КТ.

### 4. Чистые справочники
Названия, коды, допустимые материалы — централизованы, без дублирования.

### 5. Масштабируемость
Легко добавлять новые типы фактов, показателей, исследований.

## Интеграция

- Хранение ID ЕМИАС у пациента
- Возможность импорта данных пациента
